goto_rt3 Module

This node recive the user input from ui_rt3 and relay the position to move_base. Also, when the action return the status, this node will print out the result.

Action(s): /move_base

Service(s): /activate_goto

scripts.goto_rt3.activate_ = False

state of node (True/False). True means the node is enabled to control the robot goal.

Type

Boolean

scripts.goto_rt3.client_ = None

Client side service handeler (between ui_rt3 and goto_rt3

scripts.goto_rt3.done_cb(status, result)[source]

Function for printing the state of move_base Args: status(int): State of move_base result(int): Result of move_base Returns: -

scripts.goto_rt3.go_to_switch(req)[source]

Function for enable/disable this module Args: req(ROS service): State of goto_rt3 Returns: res(ROS service): Response msg

scripts.goto_rt3.main()[source]

This function initializes the ROS node action and service. After initialization, this function waiting for the trigger from ui_rt3 (via service (activate_goto)) Then, starting relay the information to move_base. The user message is passed to the service move_base.
